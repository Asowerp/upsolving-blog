## D. Integer-duplicated Path

### 题意

给定一棵树，树上有 $N$ 个顶点，编号为 $1,2,\dots,N$ 。第 $i$ 条边连接顶点 $U_i$ 和 $V_i$ 。
每个顶点 $i$ 上都写有一个整数 $A_i$ 。

针对所有 $k=1,2,\dots,N$ 回答下面的问题。

- 问题：在从顶点 $1$ 到顶点 $k$ 的简单路径（不多次访问同一顶点的路径）上的顶点中，如果存在两个写有相同整数的不同顶点，则输出 "是"；否则，输出 "否"。

### 思路

简单 dfs。注意到从 1 到 $k$ 的简单路径的遍历和我们 dfs 遍历是等价的，我们存一下当前路径上出现的数，通过一次 dfs 就能解决问题。

时间复杂度 $O(n)$。空间复杂度 $O(n)$。

### Code

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int)(x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 2e5 + 5;
unordered_map<int, int> s;
int n;
int a[MAXN];
int ans[MAXN];
vi adj[MAXN];

void dfs(int u, int fa, bool ok) {
    if (ok)
        ans[u] = 1;
    else if (s.contains(a[u]))
        ans[u] = 1, ok = true;
    s[a[u]]++;
    for (int v : adj[u]) {
        if (v == fa)
            continue;
        dfs(v, u, ok);
    }
    if (--s[a[u]] == 0)
        s.erase(a[u]);
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n;
    for (int i = 1; i <= n; i++)
        cin >> a[i];
    for (int i = 1; i < n; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].emplace_back(v), adj[v].emplace_back(u);
    }
    dfs(1, 0, 0);
    for (int i = 1; i <= n; i++) {
        if (ans[i])
            cout << "Yes\n";
        else
            cout << "No\n";
    }
    return 0;
}
```

## E. Simple Division

这题唐了。

### 题意

给你整数 $N$ 和 $M$ 。求 $\lfloor N/M \rfloor$ 除以 ${10007}$ 的余数。

在这个问题中， $N$ 并非直接给出，而是以运行长度编码的形式给出。
具体来说， $N$ 由数字 $c_i$ 和整数 $l_i$ 组成的 $K$ 对序列表示。
要恢复原 $N$ ，请使用以下步骤。
- 首先，让字符串 $S$ 为空字符串。
- 对于 $i=1,2,\dots,K$ ，重复以下操作。
    - 将 $l_i$ 份数字 $c_i$ 追加到 $S$ 的末尾。
- 将最后的 $S$ 解释为一个整数；这个整数就是 $N$ 。

### 思路

第一个转化是发现要求 $\lfloor N/M \rfloor\mod 10007$ 等价于求 $N\mod M\cdot 10007$ 之后除以 $M$。

然后一个比较显然的思路是用等比数列用 $\dfrac{10^{i}-10^j}{9}$ 去解决。但是一个很残酷的事实是我们不知道 $\gcd(9,m)$ 是否为 1，也就是说我们不知道 9 的逆元是否存在。这很难办了。

所以，我们需要采用等比数列求和的另外一个想法——分治。我们把序列拆成两半之后算，这样复杂度是 $O(\log n)$。这是因为我们要计算的本质是
$$\displaystyle \sum_{i=1}^{N} c_i \times R_{l_i} \times 10^{\sum_{j=i+1}^{N} l_j}$$
其中 $R_k$ 表示 $k$ 个连续的 1。

对每一串连续 1 的我们都这么算，这样时间复杂度就是 $O(K\log L)$ 了。

```cpp
#include<bits/stdc++.h>

using namespace std;
using ll=long long;

ll mod;

ll power(ll a,ll b){ // a^b
  ll res=1,cur=a;
  while(b>0){
    if(b&1ll){
      res*=cur; res%=mod;
    }
    cur*=cur; cur%=mod;
    b>>=1;
  }
  return res;
}

int main(){
  ll k,m;
  cin >> k >> m;
  mod=(m*10007);
  vector<ll> c(k),l(k);
  for(ll i=0;i<k;i++){
    cin >> c[i] >> l[i];
  }

  vector<ll> pow_keep(30); // 10^{2^d}
  pow_keep[0]=10;
  for(ll d=1;d<30;d++){
    pow_keep[d]=(pow_keep[d-1]*pow_keep[d-1])%mod;
  }

  vector<ll> Rpow2(30);
  Rpow2[0]=1;
  for(ll d=1;d<30;d++){
    Rpow2[d]=(Rpow2[d-1]*pow_keep[d-1]+Rpow2[d-1])%mod; // 获得 2^d 个连续的 1 mod m*10007 的值
  }

  ll res=0;
  ll dgt=0;
  for(ll i=k-1;i>=0;i--){
    ll ce=(c[i]*power(10,dgt))%mod;
    ll R=0;
    for(ll d=29;d>=0;d--){
      if(l[i]&(1ll<<d)){
        R=(R*pow_keep[d]+Rpow2[d])%mod;
      }
    }
    res+=ce*R; res%=mod;
    dgt+=l[i];
  }
  cout << res/m << "\n";
  return 0;
}
```

## F. Authentic Traveling Salesman Problem

这题拉了。考场上没想出来。

### 题意

在一个二维平面上有 $N$ 个位置，编号为 $1$ 至 $N$ 。位置 $i$ 位于坐标 $(X_i, Y_i)$ 处。
你将从位置 $1$ 出发，准确地访问每个位置一次，然后返回位置 $1$ 。
从位置 $i$ 移动到位置 $j$ 需要 $\vert X_i - X_j \vert + \vert Y_i - Y_j \vert$ 秒。
输出一条从位置 $1$ 出发，在 $10^{10}$ 秒内到达所有位置并返回位置 $1$ 的路线。保证在给定的约束条件下至少存在一条这样的路线。

### 思路

很显然这是曼哈顿距离，那么一个比较显然的贪心就是按 $x$（或 $y$）排序之后蛇形走。但是很不幸 WA 了。

那么一个自然的想法是，原先我们这样在同一个位置才向上，有没有可能出现反复蛇形导致我们其实没有很优秀呢？因此我们考虑扩大一个蛇形走的范围，在一个区间内横向蛇形向上，然后向右到另一个区间，再蛇形向下。

我们下面需要分析这样子的路径长度是多少。假设块长是 $L$，那么
 - 横向向上/下：最多是 $H+n_i L$ 步，总共 $\dfrac HL$ 个块，因此是 $\dfrac{H^2}{L}+\sum n_i H=\dfrac{H^2}L+NL$ 步。
 - 走到下一个区间：最多 $2NB=2H$ 步。
 - 走到一开始：最多 $2H$ 步。
因此总共走 $4H+\dfrac{H^2}{L}+NL\gtrsim 4H+2H\sqrt N$ 步。取 $L=\dfrac{H}{\sqrt N}$ 即可。这是小于 $10^{10}$ 的。

## G. Conquest

这题神了。

### 题意

高桥和青木玩一个卡牌游戏。游戏按照以下规则进行。

- 高桥出示卡组 $A_1, A_2, \dots, A_N$。这里 $N \geq 3$。
- 青木出示卡组 $B_1, B_2, B_3$。
- 高桥从青木的卡组中选择一个卡组并禁止它。青木也从高桥的卡组中选择一个卡组并禁止它。这两个禁止操作是同时进行的。（也就是说，任何一方都不能根据另一方的选择来行动。）
- 然后，高桥和青木各自同时宣布他们将使用自己出示的卡组中的哪一个。但是，被禁止的卡组不能被宣布。
- 然后，两名玩家使用他们宣布的卡组进行恰好一次对战。结果，其中一人获胜。

给定整数 $X_{i,j}$ $(1 \leq i \leq N, 1 \leq j \leq 3)$。令 $p_{i,j} = \dfrac{X_{i,j}}{10^6}$。$p_{i,j}$ 是高桥使用卡组 $A_i$ 且青木使用卡组 $B_j$ 时，高桥的胜率。高桥和青木都事先知道 $p_{i,j}$ 的值。
当双方都采取行动以最大化自己的胜率时，求高桥的胜率。

共有 $T$ 个测试用例；请分别解决每个测试用例。

::: details 注释
当双方都采取行动以最大化自己的胜率时，高桥的胜率是多少？

在每个阶段，每个玩家根据截至该点所揭示的信息，为可用的选择分配总和为 1 的概率，并根据这些概率随机选择一个选择。

每个玩家选择一个策略，最大化*最小可能的胜率*，即当对手选择一个最小化玩家胜率的策略时的胜率。如果有多个满足此条件的策略，可以选择其中任何一个。

当双方都根据这个原则行动时，无论选择的具体策略如何，高桥的胜率都是唯一确定的。请找出这个值。
:::

数据范围：

-   $1 \leq T \leq 500$
-   $3 \leq N \leq 5 \times 10^4$
-   $0 \leq X_{i,j} \leq 10^6$

### 背景知识

这题考场上没做出来。真的神了这道题。

首先注意到这是个博弈论问题。然后我没有怎么接触过博弈论，这不完蛋了吗。那刚好以此为契机学习一下相关知识。

一些预备知识如下。

::: tip 双人零和博弈
符合以下规则的博弈被称为**双人零和博弈**：
- 有两位玩家。
- 每位玩家恰好进行一次选择，其结果决定了每位玩家的收益。
- 每位玩家在决定自己的行动前，不知道对手的行动。
- 每位玩家可选择的行动数量是有限的。
- 博弈结果产生的两位玩家的收益之和为 $0$。
:::

像原始问题这种每位玩家都试图最大化自己获胜概率的博弈中，如果我们将他们的获胜概率定义为收益函数，那么两者之和为 $1$，因此并非零和；然而，由于总和是常数，我们可以安全地将其重新解释为一个双人零和博弈。这等价于在说，零和游戏可以视为常和游戏的特殊情形。这是因为，任何常和游戏都可以通过对某一方的收益整体加上或减去一个常数，等价地转化为零和游戏，所以仅需要讨论零和游戏。

另外，原始问题有两次行动阶段（禁用阶段和选择牌组阶段），但每个阶段的选择是同时进行的，因此可以从后一阶段开始回溯分析博弈，并且可以应用相同的讨论。像这样，两个人同时进行操作的博弈就是零和游戏当中的**同时零和博弈**。

接下来，我们将介绍双人零和博弈（以下简称零和博弈）的性质。

在**大多数**编程竞赛的博弈论问题中，玩家选择**单个**能最大化胜率的行动通常是最优的。这被称为**纯策略**。然而，在零和博弈中，纯策略并不一定能最大化胜率，因为对手的行动是未知的。相反，让我们考虑这样一种策略：为多个行动分配概率，然后根据这些概率随机选择一个。这被称为**混合策略**。

因此，注释的第一行——

> 在每个阶段，每位玩家根据截至该阶段所揭示的信息，为可行的选择分配一组总和为 $1$ 的概率，并依据这些概率随机选择一个选择。

——描述的正是混合策略。从现在开始，我们将把混合策略简称为“策略”。

注释的第二行描述的是所谓的「最大最小策略」。我们不妨定义
$$
\Delta_n = \left\lbrace x=(x_1, x_2, \dots, x_n)^\top\ \Big\vert\ \sum_{i=1}^n x_i = 1, x_i \geq 0 \right\rbrace.
$$
换句话说，$\Delta_n$ 是分配给 $n$ 个选择的混合策略的集合。定义收益矩阵 $A$ 为一个矩阵，其第 $(i, j)$ 个分量是当 Alice 选择行动 $i$，Bob 选择行动 $j$ 时 Alice 的获胜概率。

现在我们需要解决的问题就是，给定收益矩阵 $A$，如何确定两名玩家的最优策略和最大收益？

当一个零和博弈的收益矩阵由矩阵 $A$ 表示时，定义收益函数 $E(x, y)$（其中 $x \in \Delta_n, y \in \Delta_m$）为当 Alice 和 Bob 分别选择混合策略 $x$ 和 $y$ 时 Alice 的期望收益。因此有：
$$
E(x,y)=x^\top A y.
$$
熟悉的读者可能已经发现这个具有双线性形式。并且下文中，如不特别说明，默认 $x,y$ 分别表示 Alice 和 Bob 的策略。

基于这个公式，考虑 Alice 的策略，该策略最大化「可能的最低胜率」。当对手选择最小化该玩家胜率的策略时，获胜概率为 $\min_{y\in\Delta_m} E(x,y)$。因此，形式化来讲最大化这个值的策略，以及最大收益，分别是 $\arg\max_{x}\min_y E(x,y)$ 与 $\max_x\min_y E(x,y)$，其中 $\arg\max_P f(P)$ 表示当 $f$ 取得极大值的时候 $P$ 的集合。这样的策略称为**极大化极小策略**。

当 Bob 也按照同样的原则行动时，我们需要的就是 $\max_y\min_x -E(x,y)$，但这个值，根据零和博弈的性质，等于 $-1$ 乘以 Alice 的收益 $\min_y\max_x E(x,y)$。去掉负号后，这种形式更容易处理，因此我们将使用这个表达式来考虑 Bob 的策略。

简而言之，

- Alice 选择 $\arg\max_x \min_y E(x, y)$，
- Bob 选择 $\arg\min_y \max_x E(x, y)$。

von Neumann 指出，这两个值，也就是 $\max_x \min_y E(x, y)$ 与 $\min_y \max_x E(x, y)$，是相同的。

用更加形式化的语言来写，就是说，允许混合策略的同时零和游戏中，如果双方都采取最优策略，那么，Alice 的最大收益为
$$
E=\max_{x\in\Delta_n} \min_{y\in\Delta_m} E(x, y)=\min_{y\in\Delta_m} \max_{x\in\Delta_n} E(x, y).
$$
Bob 的最大收益为 $-E$。

::: details 证明细节
首先，将 Alice 的策略表示为一个线性规划问题。我们要求解的是：

$$
\max_x \min_y x^{\top} A y.
$$

对于固定的 $x$，使 $x^{\top} A y$ 最小的 $y$ 总是纯策略（因为它是关于 $y$ 线性的）。因此我们得到：

$$
 \begin{aligned} \max. & & \min_j \sum_{i=1}^n A_{i,j} x_i \\ \mathrm{s.t.} & & \sum_{i=1}^n x_i = 1 \\ & & x_i \geq 0. \end{aligned}
$$

令 $\min_j \sum_{i=1}^n A_{i,j} x_i = v$，得到：

$$
 \begin{aligned} \max. & & v \\ \mathrm{s.t.} & & v - \sum_{i=1}^n A_{i,j} x_i \leq 0 & & (\forall j) \\ & & \sum_{i=1}^n x_i = 1 \\ & & x_i \geq 0. \end{aligned}
$$

取此线性规划的对偶。依次分配变量 $y_j$ 和 $w$，得到：

$$
 \begin{aligned} \min. & & w \\ \mathrm{s.t.} & & w - \sum_{j=1}^m A_{i,j} y_j \geq 0 & & (\forall i)\\ & & \sum_{j=1}^m y_j = 1\\ & & y_j \geq 0. \end{aligned}
$$

由于我们可以设 $w = \max_i \sum_{j=1}^m A_{i,j} y_j$，消去 $w$ 得到：

$$
 \begin{aligned} \min. & & \max_i \sum_{j=1}^m A_{i,j} y_j \\ \mathrm{s.t.} & & \sum_{j=1}^m y_j = 1\\ & & y_j \geq 0, \end{aligned}
$$

这正好匹配了 $\min_y \max_x x^{T} A y$！

基于编程竞赛的知识，一个有界可行线性规划及其对偶具有相同的最优解；因此可得：

$$
\max_x \min_y E(x, y) = \min_y \max_x E(x, y).
$$

当双方都按照此原则行动时，胜率是唯一确定的，与所选择的具体策略无关。我们接下来证明之。

设 $x^{\ast}$ 和 $y^{\ast}$ 分别是原始问题和对偶问题的任意一个最优解。（这里的“任意一个”对应着“与策略无关”。）另设 $V$ 为该线性规划的最优值。那么

-   $\forall j, \sum_{i=1}^n A_{i,j} {x_i}^{\ast} \geq V \to A^{T}x^{\ast} \geq V \mathbf{1}$，
-   $\forall i, \sum_{j=1}^m A_{i,j} {y_j}^{\ast} \leq V \to Ay^{\ast} \leq V \mathbf{1}$。

分别从左边乘以 $(y^{\ast})^T$ 和 $(x^{\ast})^T$，得到：

$$
(y^{\ast})^T A^{T}x^{\ast} \geq (y^{\ast})^T V \mathbf{1} = V,
$$

$$
(x^{\ast})^T Ay^{\ast} \leq (x^{\ast})^T V \mathbf{1} = V.
$$

因此，

$$
V \leq (x^{\ast})^T Ay^{\ast} \leq V \iff (x^{\ast})^T Ay^{\ast} = V,
$$

这意味着无论最优解如何选择，最优值是唯一的。

作为补充说明，可以相对容易地证明，对于任意函数 $f(x, y)$，都有

$$
\max_x \min_y f(x, y) \leq \min_y \max_x f(x, y).
$$

这是因为 $\forall x, \min_y f(x, y) \leq f(x, y)$ 且 $\forall y, f(x, y) \leq \max_x f(x, y)$，我们得到

$$
\forall x, \forall y, \min_{y'} f(x,y') \leq \max_{x'} f(x',y)
$$

对右边关于 $x$ 取最大值，对左边关于 $y$ 取最小值，即得所证不等式。
:::

下面的一个概念就是纳什均衡（严格来说，是混合策略纳什均衡，或称混合纳什均衡）。

称策略对 $(x^{\ast}, y^{\ast})$ 是纳什均衡，如果其满足：$\forall x, E(x^{\ast}, y^{\ast}) \geq E(x, y^{\ast})$ 且 $\forall y, E(x^{\ast}, y^{\ast}) \leq E(x^{\ast}, y)$。

通俗地说，可以解释如下：

- 当 Bob 的策略是 $y^{\ast}$ 时，Alice 没有动机将自己的策略从 $x^{\ast}$ 改变为其他策略。
- 当 Alice 的策略是 $x^{\ast}$ 时，Bob 没有动机将自己的策略从 $y^{\ast}$ 改变为其他策略。

纳什均衡在某种意义上是一种最自然的定义，因为它意味着每个玩家都在按照自己的最优策略行动，并且没有动机去改变策略。
这里，在零和博弈中有以下事实：

对于策略对 $(x^{\ast}, y^{\ast})$，以下条件是等价的：

1. $(x^{\ast}, y^{\ast})$ 是一个纳什均衡。
2. $x^{\ast}$ 是最大化 $\min_y E(x, y)$ 的 $x$，而 $y^{\ast}$ 是最小化 $\max_x E(x, y)$ 的 $y$。

证明暂时略去。

### 思路

下面我们讨论原题的解法。定义 $V_{i,j}$ 为当 $A_i$ 和 $B_j$ 被禁用时的最优获胜概率。$V_{i,j}$ 是由一个 $(N-1) \times 2$ 矩阵表示的零和博弈的最优解。一旦我们得到 $V_{i,j}$ 的值，最初的禁用阶段就可以被解释为一个由 $3 \times N$ 矩阵表示的零和博弈。因此，该问题可以通过以下两个步骤解决：

1. 计算所有的 $V_{i,j}$。
2. 求解由 $3 \times N$ 矩阵表示的零和博弈。

#### 第一部分

对于第一步，我们将对每个 $j$ 独立地求解。不失一般性我们先让 $j$ 固定为 3 并将 $V_{i,j}$ 重写为 $v_i$。我们需要的是 $v_1, v_2, \dots, v_N$。设 $x$ 表示青木使用 $B_1$ 的概率。那么

$$
 \begin{aligned} v_i &= \min_{0 \leq x \leq 1} \max_{k\neq i} p_{k,1} x + p_{k,2} (1-x) \\ &= \min_{0 \leq x \leq 1} \max_{k\neq i} (p_{k,1} - p_{k,2}) x + p_{k,2}, \end{aligned}
$$

所以，如果我们定义一条直线 $L_k$ 为 $y = (p_{k,1}-p_{k,2}) x + p_{k,2}$，$v_i$ 就是在区间 $0 \leq x \leq 1$ 内，由除了 $L_i$ 之外的 $(N-1)$ 条直线构成的上包络线上的最小值。我们发现这转化成为了一个维护线段集合的问题。

::: details 官方解法
官方的一个解法是源于包络线的性质。

首先，找出由 $L_1, L_2, \dots, L_N$ 形成的上包络线。在这里，取上包络线在 $0 \leq x \leq 1$ 内取最小值的任意一个点，记作 $(x_0, y_0)$。那么有以下事实：

存在一个大小至多为 $2$ 的直线集合 $S$，使得如果 $L_k \not \in S$，则 $v_k = y_0$。

例如，如果只有两条直线 $L_i$ 和 $L_j$ 经过 $(x_0, y_0)$，那么可以证明 $S = \lbrace L_i, L_j \rbrace$。（我们在此省略证明，但画图可能有助于直观理解。）即使有三条或更多直线经过 $(x_0, y_0)$，或者取最小值的 $x$ 坐标形成一个线段，或者 $x_0=0$ 或 $x_0=1$，我们仍然可以证明除了至多两条直线外，$v_k = y_0$ 成立。

利用这个事实，下面的方法可以在 $O(N \log N)$ 时间内解决问题：

1. 找出 $(x_0, y_0)$ 和 $S$。
2. 对于 $i \in S$，单独计算 $v_i$，对于其他的 $v$，令 $v=y_0$。

但是，这个解法容易出错，因此精确的实现需要妥善处理多个步骤中出现的计算误差，所以强制使用此解法需要对计算几何有相当的了解。另外，在我们的问题中，所有坐标都可以用有理数计算，使用基于有理数的几何库也是一种选择。
:::

在此，我们讲解一种基于李超树的做法。

首先，什么是李超树？

李超树解决的问题是，在二维平面上不断加入一次函数 $y=ax+b$，并且快速查询单点 $y$ 的最大值。换言之，求上包络线在给定点处的值。

李超树本质上是一棵线段树，它的每个节点代表一个横坐标区间 $[l, r]$。
与普通线段树不同，李超树的每个节点只记录一条直线。这条直线是该区间**中点** $mid = \lfloor \frac{l+r}{2} \rfloor$ 处 $y$ 值最大的那条直线（我们称之为 **优势直线**）。

1. 插入直线：当你要在这个区间 $[l, r]$ 插入一条新直线 $f_{new}$ 时，拿它与当前节点记录的老直线 $f_{old}$ 比较：
    - 计算它俩在中点 $mid$ 的函数值。
    - 谁在中点更大，谁就留在当前节点，成为新的 $f_{old}$。
    - 输掉的那条直线（在中点较小的直线），有没有可能在区间的左半边或右半边比赢家更大呢？有可能！因为两条直线可能会相交。
    - 我们比较它们在左端点 $l$ 和右端点 $r$ 的大小。如果输掉的直线在左半边某处更高，我们就把它**递归地塞进左子树** $[l, mid]$；如果在右半边更高，就**递归塞进右子树** $[mid+1, r]$。
2. 查询最大值：要查询某点 $x$ 的最大 $y$ 值，就像线段树单点查询一样，从根节点走到包含 $x$ 的叶子节点。
一路上，你会遇到很多节点，每个节点都存了一条“优势直线”。你把沿途遇到**所有直线的 $f(x)$ 都算一遍，取最大值**，就是最终答案。

时间复杂度：插入和查询都是 $O(\log L)$，其中 $L$ 是区间长度。

普通的李超树处理的是**整数域**。但在本题中，$x$ 表示概率，定义域是实数区间 $[0, 1]$。我们无法用普通的整数线段树。

于是我们可以借鉴动态开点线段树的思想。把根节点代表的区间设为 $[0.0, 1.0]$。中点 $mid = 0.5$。每次递归，区间减半。由于实数域可以无限细分，如果不加以限制，递归会永远进行下去（或者直到浮点数精度爆炸）。因此，我们人为设定一个**最大深度 $D$**（比如 $D=50$）。当递归深度达到 $D$ 时，强制停止并丢弃那条较弱的直线。因为每次区间减半，到了第 50 层，区间的长度已经是 $2^{-50}$，加上斜率限制在 $[-1, 1]$ 之间，此时舍弃直线造成的误差上限不超过 $2^{-50}$，对于本题的 $10^{-6}$ 精度要求完全足够。一开始树是空的，只有当一条直线被推到某个子区间时，我们才在内存中创建出那个子节点，以此节省空间。

在原题中，我们需要求的是 **除了 $L_i$ 之外**的 $(N-1)$ 条直线的上包络线。
如果每次遇到一个 $i$ 都要重新建一棵包含 $(N-1)$ 条直线的树，时间复杂度会爆炸。采用前后缀分解的思想，我们需要一种方法，能快速得到前 $i-1$ 条直线和后 $N-i$ 条直线构成的树。

回想线段树，我们在遇到这样子一个节点一个节点的静态加入查询的时候，引入了所谓的可持久化线段树，也就是主席树，我们能不能做到一个类似的数据结构呢？当然可以。当我们在树中插入一条新直线时，我们**不修改原有的节点，而是创建被修改节点的副本**。

注意到仅仅维护一棵树不够用（因为前缀信息发生了覆盖），所以我们需要维护两棵树，分别维护李超树的**前缀**版本和**后缀**版本：
- $s_0$：空树
- $s_1$：在 $s_0$ 基础上插入 $L_1$
- $s_2$：在 $s_1$ 基础上插入 $L_2$
...
- $s_i$：包含了 $L_1, \dots, L_i$ 的李超树版本。

同理，反向建树得到后缀版本：
- $t_{N+1}$：空树
- $t_i$：包含了 $L_i, \dots, L_N$ 的李超树版本。

现在，回到我们的公式：
$$v_i = \min_{0 \leq x \leq 1} \max_{k\neq i} f_k(x)$$
其中 $f_k(x) = (p_{k,1} - p_{k,2}) x + p_{k,2}$，对应的也就是直线 $L_k$。

现在我们要找**除了 $L_i$ 外的直线，在 $x$ 处的最大值**，这非常简单：
只需要分别在“前缀树 $s_{i-1}$”和“后缀树 $t_{i+1}$”中查询 $x$ 处的最大值，然后取两者的较大者即可：
$$ \text{Envelope}_{\sim i}(x) = \max \big( \text{Query}(s_{i-1}, x), \text{Query}(t_{i+1}, x) \big) $$

**如何求最小值？**
现在，我们有了一个能在 $O(D)$ 时间内计算 $\text{Envelope}_{\sim i}(x)$ 的黑盒。我们要在这个黑盒中寻找 $x \in [0, 1]$ 使得结果最小。

根据数学性质，多条一次函数的上包络线必定是一个下凸函数。对于在区间 $[0, 1]$ 内的下凸函数求最小值，完美的算法是**三分查找**。所以我们可以设计出下面的算法流程：
1. 计算每条直线 $L_k$ 的斜率和截距。
2. 从左到右构建可持久化动态李超树的前缀版本 $s_1, s_2, \dots, s_N$。
3. 从右到左构建后缀版本 $t_N, t_{N-1}, \dots, t_1$。
4. 对每个被剔除的行 $i \in \{1, 2, \dots, N\}$：
   - 初始化三分查找区间 $[L, R] = [0.0, 1.0]$。
   - 循环 $S$ 次（例如 $S = 60$ 步，满足精度）：
     - $m_1 = L + (R - L) / 3$
     - $m_2 = R - (R - L) / 3$
     - 计算 $val_1 = \max( \text{Query}(s_{i-1}, m_1), \text{Query}(t_{i+1}, m_1) )$
     - 计算 $val_2 = \max( \text{Query}(s_{i-1}, m_2), \text{Query}(t_{i+1}, m_2) )$
     - 如果 $val_1 > val_2$，说明最小值在右侧，令 $L = m_1$。
     - 否则，最小值在左侧，令 $R = m_2$。
   - 循环结束后，$v_i$ 的值即为 $val_1$（或最终的 $L$ 对应的函数值）。

所以第一步为了得到 $V_{i,j}$，我们的时间复杂度有以下三部分贡献：
- **建树**：插入 $N$ 条直线，每次插入耗时 $O(D)$，总时间 $O(ND)$。空间也是 $O(ND)$，因为可持久化每次产生 $D$ 个新节点。
- **查询**：对每个 $i$（共 $N$ 个），进行步数为 $S$ 的三分查找。每次查找查询 2 次前缀树、2 次后缀树，每次查询耗时 $O(D)$。总时间 $O(N \cdot S \cdot D)$。

其中 $N$ 是行数，$D \approx 50$ 是树深度，$S \approx 60$ 是三分步数。然后对三个 $j$ 跑三次。

#### 第二部分

现在第二部分我们可以着手转化为一个计算几何的问题。具体来讲，我们设计一个半平面交来设计一个 $O(N)$ 的 `checker` 从而进行二分。

本题最终要求解一个 $3 \times N$ 的零和博弈。设收益矩阵为 $V$，其中 $V_{i,j}$ 表示高桥选择策略 $i$（即禁用 $A_i$）、青木选择策略 $j$（即禁用 $B_j$）时高桥的胜率。我们的目标是求出博弈的值 $t^*$，即高桥在双方最优混合策略下的期望胜率。

根据博弈论，该值可以通过以下线性规划（对偶形式）求得：
存在混合策略 $q = (q_1, q_2, q_3)$（对应青木选择三种禁用方案的概率），使得对于高桥的每一个可行策略 $i$，其期望收益至少为 $t$，并且我们要最大化这个 $t$：
$$
\begin{aligned}
\text{maximize } & t \\
\text{subject to } & \sum_{j=1}^3 V_{i,j} q_j \ge t, \quad \forall i = 1 \dots N \\
& q_1 + q_2 + q_3 = 1 \\
& q_j \ge 0, \quad j = 1,2,3
\end{aligned}
$$

这引导我们使用 **二分答案** 的方法：将求最大 $t$ 的问题转化为一系列判定问题——对于给定的 $t$，判断是否存在一个混合策略 $q$ 使得所有约束成立。

对于给定的 $t$，我们需要判断是否存在 $q \in \Delta_3$ 满足：
$$
V_{i,1} q_1 + V_{i,2} q_2 + V_{i,3} q_3 \ge t, \quad \forall i
$$
利用 $q_3 = 1 - q_1 - q_2$，将三维变量 $(q_1, q_2, q_3)$ 降至二维 $(q_1, q_2)$。约束条件也随之变换：

1.  **概率单纯形约束**：
    $$
    q_1 \ge 0, \quad q_2 \ge 0, \quad q_1 + q_2 \le 1
    $$
    这定义了一个二维区域——**单位三角形** $\Delta$，顶点为 $(0,0), (1,0), (0,1)$。

2.  **收益不等式约束**（代入 $q_3$）：
    $$
    V_{i,1} q_1 + V_{i,2} q_2 + V_{i,3} (1 - q_1 - q_2) \ge t
    $$
    整理得：
    $$
    (V_{i,1} - V_{i,3}) q_1 + (V_{i,2} - V_{i,3}) q_2 \ge t - V_{i,3}
    $$
    令：
    $$
    a_i = V_{i,1} - V_{i,3}, \quad b_i = V_{i,2} - V_{i,3}, \quad c_i = t - V_{i,3}
    $$
    则每个约束 $i$ 变为一个二维**半平面**：
    $$
    a_i q_1 + b_i q_2 \ge c_i
    $$

于是，判定问题等价于：**单位三角形 $\Delta$ 与所有半平面 $H_i: a_i x + b_i y \ge c_i$ 的交集是否非空？**

我们可以用多边形裁剪算法（如 Sutherland–Hodgman 算法）来高效地检查交集是否非空。

1.  **初始化**：将当前多边形 $P$ 设为单位三角形 $\Delta$，顶点列表为 $[(0,0), (1,0), (0,1)]$。
2.  **依次裁剪**：对于每个半平面 $H_i$，用它对当前多边形 $P$ 进行裁剪，得到新多边形 $P'$。
    *   **裁剪规则**：对于多边形 $P$ 的每条边 $AB$（$A$ 到 $B$），判断点 $A$ 和 $B$ 相对于半平面 $a_i x + b_i y \ge c_i$ 的位置。
        *   点 $P$ 在**内侧**（满足不等式）的条件：$a_i P_x + b_i P_y \ge c_i - \epsilon$（$\epsilon$ 是一个极小的容差，如 $10^{-12}$，用于处理浮点误差）。
    *   根据 $A$ 和 $B$ 的内/外侧状态，将顶点加入新多边形：
        *   $A$ 在内，$B$ 在内：加入 $B$。
        *   $A$ 在内，$B$ 在外：加入线段 $AB$ 与半平面边界的交点。
        *   $A$ 在外，$B$ 在内：加入交点，再加入 $B$。
        *   $A$ 在外，$B$ 在外：不加点。
    *   **交点计算**：需要计算直线 $AB$ 与直线 $a_i x + b_i y = c_i$ 的交点。注意处理 $a_i$ 和 $b_i$ 同时为零的特殊情况。
3.  **提前终止**：如果在任何一次裁剪后，多边形 $P'$ 变为空（顶点数少于 3），则立即返回 `false`（不可行）。
4.  **成功**：如果顺利处理完所有 $N$ 个半平面后多边形仍非空，则返回 `true`（可行）。

**复杂度分析**：每次裁剪的复杂度与当前多边形的顶点数 $m$ 成正比。在最坏情况下，顶点数可能增长，但由于初始形状是简单的三角形，且每次裁剪是线性的，处理所有 $N$ 个半平面的总复杂度为 $O(N + \text{总顶点数})$。在实际运行中，顶点数通常保持在常数级别（因为每个半平面最多增加一个顶点），因此总复杂度为 $O(N)$。

这样，我们就完成了这道题。

总共的时间复杂度是 $O(NSD)$，瓶颈在第一部分。

一个 AI 生成的可 AC 这道题目的代码如下。

```cpp
#include <algorithm>
#include <iomanip>
#include <iostream>
#include <vector>

using namespace std;

// 1.2e7 * 20 bytes (2 int + 2 double) approx 240MB.
// 调整为 8e6，并在每个 j 循环结束后重置，单次最大仅需约 4e6 节点。
const int MAX_NODES = 8000005;
const double EPS = 1e-12;

struct Line {
    double a, b;
    double get(double x) const { return a * x + b; }
};

struct Node {
    Line line;
    int l, r;
} tr[MAX_NODES];

int node_cnt = 0;

// 新建节点，初始化默认值
int new_node() {
    if (++node_cnt >= MAX_NODES) return 0; // 这里的安全检查防止越界
    tr[node_cnt].l = tr[node_cnt].r = 0;
    tr[node_cnt].line = {0, -1e18}; // 极小值
    return node_cnt;
}

int insert(int u, double l, double r, Line L) {
    int v = new_node();
    if (u)
        tr[v] = tr[u]; // 可持久化拷贝
    else
        tr[v].line = {0, -1e18};

    double mid = (l + r) / 2.0;
    bool left_better = L.get(l) > tr[v].line.get(l);
    bool mid_better = L.get(mid) > tr[v].line.get(mid);

    if (mid_better) swap(tr[v].line, L);
    if (r - l < 1e-13 || !L.a && L.b < -1e17)
        return v; // 精度够了或线太烂

    if (left_better != mid_better)
        tr[v].l = insert(tr[v].l, l, mid, L);
    else
        tr[v].r = insert(tr[v].r, mid, r, L);
    return v;
}

double query(int u, double l, double r, double x) {
    if (!u) return -1e18;
    double res = tr[u].line.get(x);
    if (r - l < 1e-13) return res;
    double mid = (l + r) / 2.0;
    if (x <= mid)
        res = max(res, query(tr[u].l, l, mid, x));
    else
        res = max(res, query(tr[u].r, mid, r, x));
    return res;
}

// 计算几何部分
struct Point {
    double x, y;
};

vector<Point> clip(const vector<Point> &poly, double a, double b,
                   double c) {
    vector<Point> res;
    int n = poly.size();
    for (int i = 0; i < n; i++) {
        Point p1 = poly[i], p2 = poly[(i + 1) % n];
        double d1 = a * p1.x + b * p1.y - c;
        double d2 = a * p2.x + b * p2.y - c;
        if (d1 >= -EPS) {
            res.push_back(p1);
            if (d2 < -EPS) {
                double t = d1 / (d1 - d2);
                res.push_back({p1.x + t * (p2.x - p1.x),
                               p1.y + t * (p2.y - p1.y)});
            }
        } else if (d2 >= EPS) {
            double t = d1 / (d1 - d2);
            res.push_back(
                {p1.x + t * (p2.x - p1.x), p1.y + t * (p2.y - p1.y)});
        }
    }
    return res;
}

void solve() {
    int N;
    cin >> N;
    vector<vector<double>> P(N + 1, vector<double>(4));
    for (int i = 1; i <= N; i++)
        for (int j = 1; j <= 3; j++) {
            int x;
            cin >> x;
            P[i][j] = x / 1e6;
        }

    vector<vector<double>> V(N + 1, vector<double>(4));
    for (int j = 1; j <= 3; j++) {
        int c1 = (j == 1 ? 2 : 1), c2 = (j == 3 ? 2 : 3);

        node_cnt = 0; // 每个 j 重置一次，极大节省内存
        vector<int> pref(N + 2, 0), suff(N + 2, 0);

        for (int i = 1; i <= N; i++)
            pref[i] = insert(pref[i - 1], 0, 1,
                             {P[i][c1] - P[i][c2], P[i][c2]});
        for (int i = N; i >= 1; i--)
            suff[i] = insert(suff[i + 1], 0, 1,
                             {P[i][c1] - P[i][c2], P[i][c2]});

        for (int i = 1; i <= N; i++) {
            double L = 0, R = 1;
            for (int k = 0; k < 60; k++) {
                double m1 = L + (R - L) / 3, m2 = R - (R - L) / 3;
                auto f = [&](double x) {
                    return max(query(pref[i - 1], 0, 1, x),
                               query(suff[i + 1], 0, 1, x));
                };
                if (f(m1) > f(m2))
                    L = m1;
                else
                    R = m2;
            }
            auto f = [&](double x) {
                return max(query(pref[i - 1], 0, 1, x),
                           query(suff[i + 1], 0, 1, x));
            };
            V[i][j] = f(L);
        }
    }

    double low = 0, high = 1;
    for (int k = 0; k < 25; k++) {
        double mid = (low + high) / 2;
        vector<Point> poly = {{0, 0}, {1, 0}, {0, 1}};
        bool ok = true;
        for (int i = 1; i <= N; i++) {
            poly = clip(poly, V[i][1] - V[i][3], V[i][2] - V[i][3],
                        mid - V[i][3]);
            if (poly.empty()) {
                ok = false;
                break;
            }
        }
        if (ok)
            low = mid;
        else
            high = mid;
    }
    cout << fixed << setprecision(10) << low << "\n";
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    int T;
    cin >> T;
    while (T--) solve();
    return 0;
}
```
